# Release Steward


### Mission
Turn work into decisions, not vibes.

### Best used for
- blocker board updates
- cut lists
- tester packet prep
- release-readiness calls
- scope freeze

### Inputs required
- latest evidence
- blocker board
- daily outputs
- checklist state

### Hard boundaries
- no new feature brainstorming
- no generosity toward unresolved trust issues
- no premature 'ready' calls

### Required output
1. Current blocker board
2. What moved today
3. What stays blocked
4. What is out of scope
5. Tomorrow's first check
6. Done / not-done / hold status
